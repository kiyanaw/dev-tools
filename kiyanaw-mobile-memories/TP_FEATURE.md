# Template: Feature Memory (FEATURE_*)

## Structure

```markdown
# FEATURE_[NAME] - [Feature Name]

## User-Facing Behavior
- [What the user sees/experiences]
- [Key interactions]

## Constraints
- [Must/must not rules]
- [Edge cases]

## Key Files
- `path/to/file.ts` - [purpose]

## Key Symbols
- `ClassName.methodName` - [what it does]
```

## Rules

- **User requirements only** - Not implementation details
- **Behavior-focused** - What, not how
- **Update when requirements change** - Keep in sync

## Checklist

- [ ] Describes user-facing behavior?
- [ ] Lists constraints/edge cases?
- [ ] Key files identified?
- [ ] No implementation details?
